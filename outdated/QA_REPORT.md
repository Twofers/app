# TWOFER iOS vs Android Static QA Report

Date: 2026-06-09

Scope: audit only. I did not edit app code, start Expo, run Android, run iOS, open a simulator/emulator, build, submit, publish, apply migrations, or inspect secrets. This is a read-only static review of the Expo / React Native TypeScript app and local config. Push delivery and universal links still require real TestFlight iPhone verification.

## Executive Findings

1. **Share Deal missing on iOS is most likely a build flag issue.** The only EAS profile that sets `EXPO_PUBLIC_ENABLE_SHARE_DEAL=true` is `dev-client-apk`, an Android APK profile (`eas.json:35-42`). TestFlight/production iOS profiles do not set it, so the Share Deal UI is intentionally not rendered.
2. **ZIP and birthday/age have real iOS UX bugs.** ZIP uses iOS `number-pad` while accepting ZIP+4 with a hyphen, and the iOS birthday spinner has no Done/Cancel path.
3. **iOS push is not statically proven ready.** The JS flow exists, but local config does not show an iOS push entitlement/background mode, and EAS/APNs credentials cannot be verified from this repo. The token registration also fails silently.
4. **No obvious Android-only rendered button was found.** Platform rendering branches are mostly Android hardware back handling, Android map key guards, iOS map URLs, DateTimePicker behavior, notification channels, and tab-mode hiding. Missing controls are more likely caused by feature flags, account mode, icons, or keyboard/safe-area behavior.

## Share Deal Detail

No Share Deal code is behind `Platform.OS`. The feature is controlled by runtime env:

- `isShareDealEnabled()` returns true only when `EXPO_PUBLIC_ENABLE_SHARE_DEAL === "true"` (`lib/runtime-env.ts:107-109`).
- The main detail button renders only when `shareDealEnabled` is true (`app/deal/[id].tsx:527-537`).
- The QR modal's "Send to a friend" control renders only when `onShare` is passed (`components/qr-modal.tsx:425-455`), and `onShare` is passed only when `shareDealEnabled` is true (`app/deal/[id].tsx:580-582`).
- `openShareSheet` uses React Native `Share.share` with message/url/title (`lib/share-deal.ts:121-126`). That API should exist on iOS; the first failure is that the UI probably never appears in the iOS build.

Most likely reason it does not show on iOS: the TestFlight profile inherits `preview` or `production`, neither of which sets `EXPO_PUBLIC_ENABLE_SHARE_DEAL`. Only `dev-client-apk` sets it (`eas.json:35-42`).

## Notifications Detail

What exists:

- Foreground notification handler is registered for native platforms (`app/_layout.tsx:29-51`).
- Android channel is created only on Android (`app/_layout.tsx:43-50`), which is expected.
- Permission helper calls `Notifications.requestPermissionsAsync()` (`lib/expo-notifications-support.ts:31-52`).
- Token registration calls `Notifications.getExpoPushTokenAsync({ projectId })` and upserts `push_tokens` with platform (`lib/push-token.ts:29-52`).
- App registers token on authenticated tab layout (`app/(tabs)/_layout.tsx:58-65`) and after opt-in in Home/Settings/Account (`app/(tabs)/index.tsx:458-472`, `app/(tabs)/settings.tsx:111-137`, `app/(tabs)/account.tsx:186-203`).
- Local reminders and local new-deal notifications are scheduled through safe wrappers (`lib/notifications.ts:92-113`, `lib/notifications.ts:137-231`).

What is missing or not provable from this repo:

- No local `aps-environment` string was found.
- No local `UIBackgroundModes` / `remote-notification` entry was found.
- No local APNs key, EAS credential status, provisioning profile, or Apple capability state can be verified from the files.
- `expo-notifications` plugin is present (`app.json:295`), but the iOS credential/capability side still must be checked in EAS/Apple.
- Remote push cannot be confirmed on a simulator. It must be tested on a physical iPhone with the TestFlight build.

## Recommended Fix Order

1. **Share Deal visibility:** enable `EXPO_PUBLIC_ENABLE_SHARE_DEAL=true` for the exact iOS/TestFlight build profile/environment after approval, then rebuild TestFlight.
2. **iOS push foundation:** verify APNs key, push capability, provisioning entitlement, EAS credentials, and Expo project ID; rebuild; test on a real iPhone.
3. **ZIP and birthday:** fix shared ZIP input behavior, add iOS Done/autofill, and replace the iOS birthday spinner with Done/Cancel.
4. **Settings consistency:** make settings ZIP use the same shared validation/input as onboarding/profile.
5. **iOS control visibility QA:** verify tab icons, hidden-tab mode, keyboard-covered buttons, and DateTimePicker flows on TestFlight.
6. **Permission/review copy:** align camera/photo/mic copy before App Review.
7. **Lower-risk UX polish:** numeric input Done controls, report modal keyboard behavior, redemption manual-code submit, directions fallback, location default messaging.
