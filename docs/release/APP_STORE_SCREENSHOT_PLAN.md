# App Store Screenshot Plan

Use the final TestFlight/release-candidate build. Avoid debug banners, empty inventory, billing CTAs, Stripe, pricing, placeholders, secrets, QR token text transcription, and prohibited branding terms.

## Required iPhone Screenshots

1. Consumer home with live Dallas-area offers.
2. Deal detail with business/location and clear offer terms.
3. Claim or wallet screen showing redemption readiness.
4. Map or nearby discovery with ZIP/location context.
5. Favorites or optional alerts opt-in/settings.
6. Business dashboard with sample metrics, if merchant side is included in public listing.
7. AI-assisted offer creation review screen, if merchant side is included.
8. Offer publish/review screen, if merchant side is included.

## Setup Notes

- Use seeded sample businesses and offers.
- Confirm mobile billing is hidden before capture.
- Keep the first screenshot populated, not an empty Dallas launch state.
- Capture English first. Spanish/Korean screenshots require native copy review before broader localization marketing use.
