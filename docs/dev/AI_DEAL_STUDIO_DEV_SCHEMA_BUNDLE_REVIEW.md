# AI Deal Studio Dev Schema Bundle Review

This review supports the generated bundle:

```text
docs/dev/AI_DEAL_STUDIO_DEV_SCHEMA_BUNDLE.sql
```

The bundle is generated by:

```powershell
node .\scripts\build-ai-studio-dev-schema-bundle.mjs
```

Apply it only to the separate Supabase development project. Do not apply it to production.

## Included Areas

- Auth-adjacent app profile tables and role routing support.
- Business owner account flow.
- `businesses`, `business_profiles`, `business_locations`, and business menu items.
- Business logo bucket setup and owner upload policy.
- Existing deal templates, deals, claims, public feed helpers, and claim count helpers.
- Existing `deal-photos` bucket setup and owner upload policy.
- AI generation logs, quota helper, cost ledger, and provider circuit breakers.
- Offer definition/version draft foundation.
- AI ad generation/media-library draft tables needed for future AI Deal Studio testing.
- Private `ai-deal-assets` bucket helper with owner-scoped storage RLS.

## Explicitly Excluded

- Production-hosted cron migrations.
- Paid billing, subscription history, trial, entitlement, and credit-consumption migrations.
- Production release-push cron scheduling.
- Demo content markers and QA seed deal migrations.
- Redemption-mode/staff-session migrations, because AI Deal Studio publishing remains disabled for the dev APK.
- Offer-version publish RPC migrations, because publishing remains disabled for this phase.

The bundle uses a small dev-only business-location helper instead of the historical location migration because the historical migration also introduced paid-plan columns that are not needed for this dev APK.

## Why This Is Not `supabase db push`

The normal migrations directory contains production-targeted cron and paid billing history. Running `supabase db push` from this repo would attempt to apply that history. This bundle is a reviewed allowlist instead.

## Pre-Apply Checks

From the repo root:

```powershell
Get-Content .\supabase\.temp\project-ref
node .\scripts\build-ai-studio-dev-schema-bundle.mjs
rg -n "kvodhiqhdqnptqovovia|cron\.schedule|net\.http_post|stripe|sk_live_|sk-[A-Za-z0-9]{20,}" .\docs\dev\AI_DEAL_STUDIO_DEV_SCHEMA_BUNDLE.sql
```

The linked project ref must be the dev project. The `rg` command should print no matches.

## Apply Step

Applying schema is a hard-gated database change. After explicit approval, paste the generated SQL bundle into the dev Supabase dashboard SQL editor, or apply it through a dev-only database connection string kept local.

After applying any RLS policy changes, run:

```powershell
node .\scripts\probe-rls-smoke.mjs
```

Do not deploy Edge Functions as part of this schema bundle step.
